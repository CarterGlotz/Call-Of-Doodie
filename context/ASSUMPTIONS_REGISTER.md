# Assumptions Register

### Humor-first positioning is the main reason players remember the game

- Why we believe it: the project brief, README, and feature set all emphasize comedy as the leading differentiator
- Risk if false: content effort may overinvest in jokes while underinvesting in progression or competition
- How to validate: compare retention and sharing behavior after humor-heavy versus challenge-heavy updates
- Status: active

### Challenge links and ghost competition can materially increase replayability

- Why we believe it: the game already invests in seeded runs, scorecards, and challenge links
- Risk if false: social systems may not justify their complexity
- How to validate: track reuse of challenge links and repeat sessions from competitive prompts
- Status: active

### Frontend-only architecture is still enough for the current phase

- Why we believe it: the live product works today without an active gameplay backend
- Risk if false: live features may stall without stronger server support
- How to validate: review roadmap needs against current Pages + Supabase architecture
- Status: active

### Players will tolerate high chaos if feedback stays legible

- Why we believe it: many existing features already increase enemy and perk complexity without breaking the run loop
- Risk if false: later content may tip the game into unreadable noise
- How to validate: test harder waves and event stacks for clarity and death comprehension
- Status: active
